Git - система управления версий, позволяющая разработчикам отслеживать изменения в проекте

### Первичная настройка Git:

```console
git config --global user.name "John Doe"
git config --global user.email johndoe@example.com
```
Первая команда передает имя пользователя, вторая почту. Благодаря параметру `--global` достаточно произвести настройку один раз.

### Основные команды 

`git --version` - команда для проверки установленной версии гит 

`git config --list` - выводит в консоль текущие конфигурации гита. Чтобы выйти из этой команды, нажми клавишу Q

`git init` - создает новый репозиторий в указанном каталоге

`git add` - добавляет изменения из рабочего каталога в индекс — промежуточную область между рабочей директорией и репозиторием. Это подготавливает файлы для следующего коммита.

`git commit -m "first commit"` - создается фиксация изменений (коммит). `- m "first commit"` - необязательная часть с комментарием изменения

`git remote add origin https(ссылка на репозиторий в гитхаб)` - связывает локальный и удаленный репозитории

### .gitignore. Как изолировать файлы от репозитория

1. Откройте терминал в корне вашего проекта
2. Выполните команду:

```bash
touch .gitignore
```

3. Добавьте нужные шаблоны в этот файл
4. Не забудьте закоммитить .gitignore:

```bash
git add .gitignore
git commit -m "Add .gitignore"
 ```

Работает только для еще не отслеживаемых файлов

Примеры:
```
# Игнорировать конкретную папку
node_modules/
venv/
__pycache__/

# Игнорировать все файлы с определенным расширением
*.log
*.tmp
*.cache

# Игнорировать файлы в любой папке
**/.DS_Store
**/Thumbs.db

# Игнорировать, но с исключениями
*.txt           # игнорировать все .txt файлы
!README.txt     # НЕ игнорировать README.txt

# Игнорировать по шаблону имени
temp-*
backup-*.zip

# Игнорировать только в корне
/.env
/секрет.txt

# Игнорировать папку везде
logs/
# или
**/logs/
```
